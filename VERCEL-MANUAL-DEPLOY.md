# Vercel手動デプロイガイド

## 🔍 「Redeploy」ボタンの見つけ方

VercelのDeploymentsページでは、「Redeploy」ボタンは直接表示されていません。以下の方法でアクセスできます：

### 方法1: 各デプロイの三点メニューから（推奨）

1. Deploymentsページで、**最新のデプロイ行**（`efafa1b`）の**右端にある三点メニュー（⋮）**をクリック
2. メニューから **「Redeploy」** を選択
3. 最新のコミット（`4cb263a` または `6b9ceae`）を選択してデプロイ

### 方法2: デプロイをクリックして詳細ページから

1. Deploymentsページで、**最新のデプロイ**（`efafa1b`）をクリック
2. デプロイの詳細ページが開く
3. 右上に **「Redeploy」** ボタンが表示される
4. クリックして、最新のコミットを選択してデプロイ

### 方法3: GitHubから直接デプロイをトリガー

1. GitHubリポジトリ → **Actions** タブを開く
2. Vercelのワークフローが実行されているか確認
3. 実行されていない場合は、手動でトリガー

### 方法4: Vercel CLIを使用（CLIがインストールされている場合）

```bash
# Vercel CLIをインストール（未インストールの場合）
npm i -g vercel

# ログイン
vercel login

# プロジェクトにリンク（初回のみ）
vercel link

# デプロイを実行
vercel --prod
```

## 📋 推奨される手順

1. **まず方法1を試す**（三点メニューからRedeploy）
2. **それでも見つからない場合**、方法2を試す（デプロイをクリックして詳細ページから）
3. **それでも解決しない場合**、方法4を試す（Vercel CLI）

## 💡 注意点

- 最新のコミット（`4cb263a`、`6b9ceae`）がデプロイされていないため、手動でデプロイをトリガーする必要があります
- デプロイ後、次回のpushで自動デプロイが動作するか確認してください
